## Dandan — демо интернет‑магазин видеоигр

Полноценное веб‑приложение: фронтенд (HTML + TailwindCSS + JS), бэкенд (Node.js + Express), база данных (SQLite).

### Установка и запуск

1. Установите зависимости:

```bash
npm install
```

2. Инициализируйте базу данных и демо‑данные:

```bash
npm run init-db
```

3. Запустите сервер разработки:

```bash
npm run dev
```

Откройте `http://localhost:3000` в браузере.

### Демо‑аккаунт администратора

- Email: `admin@example.com`
- Пароль: `admin`

### Стек и структура

- **Фронтенд**: `public/index.html`, `public/css/styles.css`, `public/js/app.js` — одностраничное приложение с разделами (главная, каталог, карточка игры, корзина, вход, регистрация, личный кабинет только для авторизованных). Вёрстка на TailwindCSS (CDN) + кастомный минималистичный дизайн в духе продуктовых страниц Apple.
- **Бэкенд**: `server.js`, роуты в `routes/`, контроллеры в `controllers/`, авторизация и роли в `middlewares/`.
- **БД**: SQLite (`db/database.sqlite`), схема и демо‑данные в `db/init.js`, доступ к БД через `sqlite3` и обёртку в `db/db.js` и моделях `models/`.

### Основные возможности

- Регистрация и авторизация с разделением ролей (`user`, `admin`), хеширование паролей через `bcryptjs`, сессии `express-session` + `connect-sqlite3`.
- Каталог игр с фильтрацией, поиском, сортировкой и пагинацией, все действия через Fetch API без перезагрузки.
- Страница игры с описанием, системными требованиями и встроенным трейлером (YouTube iframe).
- Корзина:
  - Гостевой режим — хранение в `localStorage`.
  - После входа локальная корзина синхронизируется с серверной (сессия).
  - Оформление заказа создаёт записи в таблицах `orders` и `order_items`.
- Личный кабинет:
  - Пользователь: редактирование профиля, история заказов.
  - Администратор: панель управления играми (добавление/редактирование/удаление, загрузка обложек через `multer` в `public/uploads`).

### Демонстрация таблиц БД

База данных хранится в файле `db/database.sqlite` и содержит 5 связанных таблиц:

- `users`
- `games`
- `genres`
- `orders`
- `order_items`

Чтобы на другом компьютере показать, что таблицы действительно есть, можно:

1. Установить любую SQLite‑утилиту, например **DB Browser for SQLite**.
2. Открыть в ней файл `db/database.sqlite`.
3. Перейти на вкладку просмотра структуры/`Browse Data` и показать список таблиц и их поля.

Либо через консольный клиент SQLite:

```bash
sqlite3 db/database.sqlite
.tables        -- список таблиц
.schema users  -- структура таблицы users
```
